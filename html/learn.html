BLOCK LEVEL VS INLINE ELEMENTS

Block level elements occupy the width of the container, forcing any following elements to the next line. Examples of inline elements are divs, paragraphs, list items etc

Inline elements occupy the width of the element and new elements are added on the same line and to the right of it. E.G span, image, links. You can't use the width property on inline elements

DIV vs SPAN, CLASS vs ID

Div and Span are html division elements, the difference is divs are block level divisions while spans are inline divisions

Classes and IDs are styling attributes used with CSS. The main difference is that IDs can also be used for in-page navigation, hence they should be used only once per page.

HTML 5 SEMANTIC TAGS

Before html 5 we basically had divs for sectioning. HTML 5 introduced more meaningful tags that are good for sectioning but also give meaning to the blocks of code and are great for SEO as well. These tags include
<header> : defines header for a document or section. The main header contains intro elements like logos, nav... pages can have multiple headers, but not in footers, headers or address
<section> : a section groups similarly themed content typically with a heading
<article> : the article is for stand alone content that can be distributed by itself
<aside> : the aside element defines some content aside from the content it is placed in (like a sidebar). The content should be indirectly related to the surrounding content
<footer> : the footer element defines a footer for a document or section
<nav> : the nav element is intended for major blocks of navigation links
<figure> : the figure tag specifies self-contained content, like illustrations, diagrams, photos, code listings etc.
<figcaption> : defines a caption for a figure element. it is placed as the first or last child of a figure element
<main> : specifies the main content of a document
<time> : defines a date/time
<details> : defines additional details that you can show or hide
<summary> : defines a visible heading for a details element
<mark> : defines marked/highlighted text
